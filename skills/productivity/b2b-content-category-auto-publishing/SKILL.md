# B2B工业品内容类目自动发布系统

## 背景
为探钱库APP建立SEO内容发布体系。每个类目（如原料、辅料、染料等）都需要：知识库 + 每日采集 + 分时段文章发布。

---

## 核心架构

每个类目需要创建7个cron任务：

```
知识库文件：~/.hermes/cache/documents/doc_[hash]_[类目名].txt
资讯文件：~/.hermes/cron/output/[类目_knowledge_YYYYMMDD].md
```

| 任务类型 | 数量 | 执行时间 | 内容 |
|---------|------|---------|------|
| 资讯采集 | 1个 | 08:00 | 联网采集→结构化知识库 |
| 文章发布 | 6个 | 8/10/12/14/16/18时 | 生成文章→配图→API发布 |

---

## 知识库结构规范

### 按B端商家工作流组织（不按产品分类）

```
一、选型知识库（开款用）
  - 选型对照表：用途→规格→优缺点→价格区间
  - 决策树：材质→尺寸→适用场景

二、验货知识库（大货用）
  - 检测标准和方法
  - AQL抽检方案
  - 国标/行标对照

三、成本核算知识库
  - 成本占比参考
  - 批发参考价（元/打、元/千个）
  - 损耗率计算

四、行业问答知识库（B端高频Q&A）
  - 模拟真实搜索词
  - 2-3句话专业回答

五、问题知识库（常见客诉与方案）
  - 问题→原因→解决方案

六、产业知识库（找货源）
  - 主要产业带分布
  - 核心供应商聚集地
```

### 资讯采集输出格式

```markdown
# [类目]知识库
> 更新日期：YYYY-MM-DD

## 【价格行情区】
| 品种 | 规格 | 价格区间 | 来源 | 日期 |

## 【选型知识库】
...

## 【验货知识库】
...

## 【SEO长尾词库】
| 关键词 | 搜索意图 | 热度 |

## 【行业Q&A】
Q: [用户真实搜索词]
A: [2-3句话回答]
```

## 文章类型规范

每个类目定义6种文章类型：

| 类型 | 侧重点 | 适用主题 |
|------|--------|---------|
| 选型指南 | 如何根据场景选对产品 | 规格对比、决策树 |
| 验货标准 | 检测方法、AQL抽检 | 国标解读、质量判定 |
| 成本核算 | 成本拆解、损耗计算 | 报价参考、预算 |
| 问题解决 | 常见问题与处理方案 | 客诉处理、工艺改进 |
| 产业分析 | 产业带、供应商评估 | 找货源、供应链 |
| 行业揭秘 | B端采购避坑 | 鉴别以次充好、套路 |

## 文章写作规范

### 标题
- 40-60字
- 含核心关键词 + 长尾词
- 具有吸引力（数字、疑问句、痛点）

### 正文
- 800-1200字
- H2/H3分段
- 反AI痕迹：禁止"总而言之"等过渡语
- 探钱库APP场景触发最多2次（不能像广告）
- 末尾有读者视角评论引导
- 无其他平台引用

### 配图
- MiniMax图生API
- prompt用英文，纯视觉场景
- 禁止任何文字/标签（会乱码）
- 中文用户友好：工厂场景、产品特写、质量检测
- 16:9比例
- 放在内容相关位置（非头尾）

## cron任务模板

### 资讯采集
```markdown
## 任务：[类目名]资讯每日采集

### 执行时间
每天 08:00

### 搜索来源
行业垂直网站优先 + 百度搜索补充

### 输出
~/.hermes/cron/output/[类目_knowledge_YYYYMMDD.md

### 格式
按上方"知识库结构规范"输出
```

### 文章发布（每个时段独立任务）
```markdown
## 任务：[类目名]文章发布-[时间]

### 执行步骤
1. 读取知识库
2. 读取最新资讯
3. 随机选1种文章类型
4. 生成800-1200字文章
5. MiniMax生成配图
6. API发布

### 输出
本地记录，不返回用户
```
